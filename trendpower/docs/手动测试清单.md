# trendpower 手动测试清单

> 一份**手动验收**清单，用来快速过一遍整套软件的核心功能是否正常。
> 约定：在一个**测试用的 git 仓库或临时目录**里跑（会真的读写文件 / 跑命令）。
> 启动：`cd 你的测试目录 && trendpower`。每条给出「操作 → 期望」，跑通就打勾。

---

## 0. 启动与配置

- [ ] **首次运行向导**：未配置模型时启动，弹出 First-Run 向导，能选 provider、填 baseURL/APIKey 并保存。
- [ ] **`/model`**：输入 `/model`，能列出已配置模型、切换默认、增删。
- [ ] **`/help`**：列出所有 slash 命令（含技能 `/coding-plan`、`/deep-research-plan`）。
- [ ] **基本对话**：输入「你好，简单介绍一下你能做什么」→ 直接给一段文字回答，**不乱调工具**。

## 1. 编辑 + 工具审批

- [ ] 输入：`在当前目录创建 hello.py，内容是打印 "hi"`。
- [ ] **期望**：模型调用 `write_file` 前**弹出审批条**（allow once / allow always / deny）。
  - 选 **deny** → 该工具被跳过，模型收到拒绝理由并据此调整。
  - 选 **allow once** → 文件被创建，内容正确。
- [ ] 追问：`把它改成打印 "hello world"` → 模型先 `read_file` 再用 `apply_patch`/`str_replace` 精确改，文件内容正确。

## 2. 多步任务 → Todo 列表

- [ ] 输入：`读一下这个项目结构，然后给 README 加一个“用法”小节，并新建一个 .gitignore`（≈3 步）。
- [ ] **期望**：右侧 **Todo 面板**出现任务清单，状态随进度从 pending → in_progress → completed 更新。

## 3. Plan 模式（只读，不改文件）

- [ ] 输入：`/coding-plan` 然后写任务，例如：`帮我把 hello.py 重构成带 main() 入口的形式`。
- [ ] **期望**：模型**只读/搜索**，最后**产出一份方案**（要改哪些文件、怎么改），**全程不实际修改文件**。
- [ ] （可选）`/deep-research-plan` + 一个调研类问题 → 只读+搜索，产出调研提纲。

## 4. 子 Agent 委派（task 工具）

- [ ] 输入：`在整个仓库里查清楚 XXX 是在哪里定义和使用的，给我结论`（XXX 换成项目里真实的函数/变量名）。
- [ ] **期望**：模型调用 **`task`（explore）** 委派一个子 Agent；流式指示器显示子任务在跑；最后**只把一句结论**并回主线（中间翻文件的噪音不进主对话）。

## 5. 反问用户（ask_user_question）

- [ ] 输入一个**故意有歧义**的需求：`给项目加个配置文件`（不说格式/位置）。
- [ ] **期望**：模型用 **ask_user_question** 弹出选择题（如 YAML/JSON/TOML），你选完后它据此继续。

## 6. MCP server 接入

- [ ] 在 `~/.trendpower/mcp_servers.json` 写入一个 stdio server（需本机有 Node/`npx`）：
  ```json
  {
    "mcpServers": {
      "filesystem": {
        "transport": "stdio",
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"]
      }
    }
  }
  ```
- [ ] 启动后输入 **`/mcp list`** → 看到 `filesystem` 已连接、列出其工具（名字带 `filesystem__` 前缀）。
- [ ] 让模型用该工具：`用 filesystem 工具列出 /tmp 下的文件` → 能调用并返回结果。
- [ ] **`/mcp reload`** → 重新连接成功。坏配置（乱填 command）→ 只报错不崩，其他 server 不受影响。
- [ ] 详细配置见 [`docs/mcp.md`](mcp.md)。

## 7. 取消 / 中断

- [ ] 给一个会跑一会儿的任务（如 `跑一下测试套件并总结结果`），中途按 **Esc / Ctrl+C**。
- [ ] **期望**：**立即停止**——正在跑的 `bash` 子进程被杀、循环中止，界面不崩，可以继续输入下一条。

## 8. 会话持久化 / 恢复

- [ ] 进行几轮对话后退出（`/exit`），重新启动，输入 **`/resume`**。
- [ ] **期望**：能选中刚才那次会话并**完整恢复**对话历史，接着聊。

## 9. 上下文压缩（可选，较难触发）

- [ ] 制造长上下文：让它**多次读大文件 / 反复跑命令**，把对话堆到很大。
- [ ] **期望**：接近阈值（默认 ~10 万 token）时，界面出现一行 `Context compacted: N → M messages …` 提示，且**对话仍能正确接续**（没忘记最初目标）。

## 10. 执行轨迹 Trace（新功能）

- [ ] 用 **`TRENDPOWER_TRACE=1 trendpower`** 启动，跑一个多步任务（最好触发并行工具 + 一个 `task` 子 Agent）。
- [ ] 退出后运行 **`trendpower trace list`** → 看到这次 run；**`trendpower trace view`** → 打印 `run → step → llm/tool` 瀑布图，含每步耗时、token 数、`✓/✗` 工具状态、子 Agent 嵌套。
- [ ] **中断测试**：跑到一半按 Ctrl+C，再 `trendpower trace view` → 该 run 标为 **aborted**（trace 文件仍可读，不丢）。

## 11. Web 实时视图（可选）

- [ ] 用 **`trendpower-web`** 启动，浏览器打开提示的 `http://127.0.0.1:8765`。
- [ ] 在终端里随便问一句触发多轮 ReAct。
- [ ] **期望**：网页实时显示每次 LLM 调用的输入/输出/工具结果（第二轮起新增内容有 ★ 高亮）；左下 **「执行轨迹 trace」面板**随每步/每个工具实时更新（run→step→llm/tool 瀑布）。

---

### 快速冒烟（只想花 5 分钟）
跑 **0（基本对话）→ 1（编辑+审批）→ 3（plan 模式）→ 4（子 Agent）→ 10（trace）** 这 5 条，就覆盖了「ReAct 循环 + 工具 + 审批 + 计划 + 隔离 + 可观测」六大主干。
