# Harness Engineering 入门：trendpower 是怎么"指挥"模型干活的

> 这篇是给**没有 AI 背景**的人看的。每出现一个专业词，我都先用大白话解释，再配一个例子。读完你会明白：一个能帮你写代码的 AI，除了"模型"本身，外面还需要一整套工程，这套工程就叫 **harness**。

---

## 一、先打个比方：模型是大脑，harness 是手脚和规矩

想象你雇了一个**很聪明但什么都不会自己动手的实习生**：

- 他脑子很好（这就是**模型**，比如 Claude、GPT、豆包）；
- 但他不会自己开电脑、不会自己翻文件、不会自己跑命令；
- 他也不知道"什么时候该停"、"危险操作要不要先问你"。

**harness（中文勉强译成"外壳/支架/驾驭层"）** 就是你给这个实习生配的：一套工具箱 + 一套工作流程 + 一套规矩。同一个实习生，配的工具和规矩好不好，决定了他是"只能聊天的玩具"还是"真能帮你干活的助手"。

trendpower 就是这样一套 harness。下面我们一个一个看它配了哪些东西、为什么这么配。

---

## 二、用一个完整例子串起来

假设你对 trendpower 说：

> "把项目里所有的 `print` 改成用 `logging`。"

一个好的 harness 会让模型这样工作（注意它是**一步一步**来的）：

1. **想**：我得先知道哪些文件里有 `print`。→ 决定用"搜索"工具。
2. **做**：调用搜索工具，得到结果："3 个文件里有 print。"
3. **想**：那我去读第一个文件看看上下文。→ 决定用"读文件"工具。
4. **做**：读文件，看到内容。
5. **想**：好，我来改。→ 决定用"改文件"工具。
6. **做**：改文件……（如此循环）
7. **想**：都改完了，没别的要做了。→ **不再调用任何工具，直接回话**："改完了，一共动了 3 个文件。"

这个"**想一步 → 做一步 → 看结果 → 再想**"的循环，是整个 harness 的心脏。它有个学名叫 **ReAct**（Reason + Act，"边推理边行动"）。

> 📍 在代码里：`trendpower-py/trendpower/agent/agent.py` 的 `Agent.stream` 就是这个循环。

下面把这个例子里出现的每个零件拆开讲。

---

## 三、零件逐个讲

### 1. Agent 循环：想一步，做一步

**是什么**：让模型"思考 → 行动 → 看结果 → 再思考"，反复进行，直到它觉得活干完了。

**为什么需要**：模型一次回话是说不完一个复杂任务的。它需要**根据每一步的结果决定下一步**——就像你修电脑，得先看哪儿坏了才知道修哪儿，不可能闭着眼睛一口气修完。

**怎么知道该停**：很简单——**当模型这一轮不再要求使用任何工具、只是正常说话时，就说明它认为干完了**，循环结束。（上面例子的第 7 步。）

**举个反例**：如果没有这个循环，模型只能回你一段文字"你应该去搜索 print，然后改成 logging"——它只会"说"，不会"做"。循环让它能真的动手。

---

### 2. 工具（Tools）：实习生的工具箱

**是什么**：一个个模型可以"调用"的功能。比如：
- `read_file`（读文件）、`write_file`（写文件）、`grep_search`（在代码里搜关键字）、`bash`（执行命令）等。
- trendpower 内置了 12 个这样的工具（在 `trendpower-py/trendpower/coding/tools/` 目录下）。

**为什么需要**：模型本身只会"输出文字"。它没法直接碰你的硬盘。工具就是它伸向真实世界的"手"。模型想读文件，它其实是输出一段"我要调用 read_file，参数是这个路径"的请求，由 harness 真正去读，再把内容递回给它。

**打个比方**：模型像一个被关在玻璃房里的专家，只能写纸条。工具就是房间里的传话筒——他写"帮我把 A 文件拿来读读"，外面的人照做，再把结果递进去。

---

### 3. 参数校验：表填错了当场打回，而不是出事

**是什么**：模型调用工具时要填"参数"（比如读哪个文件、写什么内容）。这些参数是模型生成的，**可能填错**（漏字段、路径瞎编、格式不对）。trendpower 在工具真正执行前，先用一套规则（叫 **pydantic**，一个 Python 校验库）检查参数对不对。

**为什么需要**：模型偶尔会"幻觉"——一本正经地填一个根本不存在的参数。如果不检查就执行，轻则报错崩溃，重则误删东西。

**怎么处理**：校验不通过时，**不崩溃**，而是把"你这个参数填错了，错在哪"作为结果**喂回给模型**，让它自己改正重填。

**打个比方**：实习生交表，表里有个必填项没填。你不会直接把错表拿去办事，而是退回去说"这里漏了，补一下"——他补完再交。

> 📍 在代码里：`trendpower-py/trendpower/foundation/tools.py` 的 `define_tool`。

---

### 4. 工具结果：助手汇报只说重点

**是什么**：工具执行完会有结果（比如搜索返回一长串、读文件返回一大段）。trendpower 不会把原始结果一股脑塞回给模型，而是按工具类型**整理和裁剪**后再给。

**为什么需要**：模型每次"读题"都要把之前所有内容重看一遍（这点在 [context-engineering 那篇](context-engineering-入门.md) 会细讲），所以喂回去的东西越精炼越好。

**举个例子**：
- 你让它"列出某目录下的文件"——它真正需要的是"这里有哪些文件"这个**结论**，而不是把整棵目录树一字不漏地背进脑子。所以列目录的结果会被压成一句摘要。
- 但"读文件"就不一样了——你读文件就是为了看内容，所以这个结果会保留得比较完整。

另外，每个结果还带一个**错误类型标签**（比如"文件没找到""参数非法""环境缺工具"）。这样模型一看标签就知道下一步该怎么办，而不用去猜那串报错是什么意思。

> 📍 在代码里：`trendpower-py/trendpower/agent/tool_result/policy.py`（每个工具留多少）。

---

### 5. 并行执行：一次派三个活同时干

**是什么**：如果模型一轮里同时要求"读 A、读 B、读 C"三个文件，trendpower 会让这三件事**同时进行**，而不是排队一个一个来。

**为什么需要**：省时间。三个文件串行读要等三次，同时读只等最慢的那一次。

**打个比方**：你让三个实习生分头去拿三份资料，当然是三个人同时出发，而不是一个回来了再派下一个。

> 📍 在代码里：`agent.py` 的 `_act` 用 `asyncio`（Python 的"同时干多件事"机制）实现。

---

### 6. 随时喊停（取消）

**是什么**：任务跑到一半，你想中断（比如发现它理解错了），能立刻停下来。

**为什么需要**：交互式工具必须让人能打断。如果一个慢命令要跑 5 分钟，你按了停却要等它跑完才停，那等于没法停。

**怎么做**：trendpower 用一个叫 **AbortSignal（中止信号）** 的东西，像一个全局的"急停按钮"。一按下去，正在跑的工具（比如一个 bash 命令对应的子进程）会被立刻"杀掉"。

**打个比方**：工厂流水线的红色急停按钮——一拍，所有机器立即停，而不是等这批货做完。

---

### 7. 中间件：流水线上的"检查站"

**是什么**：**中间件（middleware）** 是插在循环关键节点上的一段可插拔逻辑。trendpower 在"每次找模型之前""每次用工具之前"等时点，会依次经过一串中间件。

**为什么需要**：有些事情不属于某个具体工具，而是"每一步都要顺带做"的横切任务，比如：
- 历史太长了要压缩（见下篇）；
- 危险操作要先问用户（见下一条）；
- 把可用的"技能清单"注入进去。

把这些做成一节一节的"检查站"，而不是焊死在主循环里，好处是：**加一个新检查站很容易，去掉也很容易，互不干扰**。

**打个比方**：机场安检——值机、安检、登机口各是一个独立的检查站，旅客（也就是"这一步的请求"）依次经过。想加一道"测体温"，新设一个站就行，不用把整个流程推倒重来。

**一个重要细节**：检查站是**有顺序**的。比如"压缩历史"这个站被刻意排在最前面，因为后面的站要读历史，得先让它压实了再读。

> 📍 在代码里：`agent.py` 里那一串 `_before_model / _before_tool_use ...` 钩子。

---

### 8. 审批：危险操作要主管签字

**是什么**：像"执行命令(bash)""写文件""删改文件"这类有风险的操作，trendpower 会在执行前**弹出来问你一句"允许吗？"**，你点同意才真的做。

**为什么需要**：你不会希望一个 AI 不打招呼就在你电脑上跑命令、改文件。

**贴心之处**：你可以选"以后这个项目里都允许这个操作"，trendpower 会**记住**（存进项目设置），下次同类操作就不再打扰你。如果你**拒绝**，它不会崩，而是把"用户拒绝了"当作一条消息告诉模型，让模型换个方案或反过来问你。

**打个比方**：实习生要动公司的重要设备，得先找你签字。你可以签"这台设备以后随便他用"，省得每次都来烦你；你也可以驳回，他就得另想办法。

> 📍 在代码里：`trendpower-py/trendpower/coding/permissions/`。需要审批的工具清单在 `requires_approval.py`。

---

### 9. 子 Agent：派个实习生去查资料，只带结论回来

**是什么**：trendpower 有一个特别的工具叫 `task`。模型可以用它**派出一个"子实习生"**去独立完成一个子任务（比如"在整个项目里查清楚某个功能是怎么实现的"），这个子实习生有自己的一套独立工作空间，干完后**只把一句结论汇报回来**，中间翻了多少文件、跑了多少搜索，主线一概看不到。

**为什么需要**：这是整个设计里很关键的一招。"在项目里查个东西"经常要翻几十个文件，产生一大堆中间噪音。如果这些噪音全留在主线上，会有三个坏处：花钱（每一步都要重发，见下篇）、变慢、还把模型的注意力带偏。

**打个比方**：你要写报告，需要先查一大堆背景资料。聪明的做法是**派个实习生去图书馆查**，让他把一摞书翻完，**只给你一页摘要**——而不是把整个图书馆搬到你桌上。你的桌子（主线上下文）始终是干净的。

**两种子实习生**：
- `explore`（探索型，默认）：**只能读、不能改**，最安全，专门用来查资料/搜索。
- `general`（通用型）：可以改文件、跑命令（依然要经过上面第 8 条的审批）。

**两条安全红线**：
- 子实习生**不能再派子实习生**（防止无限套娃）。
- 子实习生**不能反过来问用户问题**（它是后台干活的，不该弹窗打扰你）。

> 📍 在代码里：`trendpower-py/trendpower/coding/tools/task.py` 和 `trendpower-py/trendpower/agent/subagent/runner.py`。

---

### 10. 软着陆：时间到了交个半成品 + 说明，而不是摔门走人

**是什么**：为了防止 AI 陷入死循环没完没了，trendpower 给每个任务设了**步数上限**（默认 100 步）。问题是：撞到上限怎么办？

**以前的做法（不好）**：直接报错崩溃，甩一句"步数用完了"，前面几分钟干的活全没了。

**现在的做法（软着陆）**：撞到上限时，不崩，而是**让模型停下来，用现有信息给你一份总结**：我做到哪一步了、还差什么、接下来建议怎么做。然后把这份总结当作回答交给你。

**为什么需要**：一个干了一半的任务，给你"进展 + 后续建议"，远比甩你一句报错有用。你可以据此接着问、或重新开一轮继续。

**打个比方**：实习生干一个大活，到下班点了还没完。好的做法是交一份"我做到这儿了，剩下这些，建议明天这么接着干"的交接说明；差的做法是直接把活一扔、人走了。

> 📍 在代码里：`agent.py` 的 `_emit_step_limit_summary`。

---

### 11. Provider（模型接口层）：插座转换头

**是什么**：trendpower 不绑定某一家模型。它把"怎么跟某家模型对话"封装成一个个 **provider（提供方适配器）**：有 OpenAI 的、有 Anthropic 的。要接豆包/火山方舟这种"兼容 OpenAI 接口"的模型，直接用 OpenAI 那个 provider 就行。

**为什么需要**：今天用这家、明天想换那家，是常事。把"对话的格式翻译"集中在 provider 这一层，**换模型时，循环、工具、规矩全都不用动**，只换一个适配器。

**打个比方**：出国旅行带的**插座转换头**——电器（你的整套 agent 逻辑）不用换，换个转换头就能插不同国家的插座（不同厂商的模型）。

> 📍 在代码里：`trendpower-py/trendpower/community/openai/` 和 `trendpower-py/trendpower/community/anthropic/`。

---

## 四、一句话总结

| 零件 | 大白话 |
|---|---|
| Agent 循环 | 想一步、做一步，直到活干完 |
| 工具 | 模型伸向真实世界的"手" |
| 参数校验 | 表填错了当场打回重填 |
| 工具结果裁剪 | 汇报只说重点，别把账册全搬上桌 |
| 并行执行 | 一次派三个活同时干 |
| 取消 | 随时能拍急停 |
| 中间件 | 流水线上一节节检查站，想加就加 |
| 审批 | 危险操作要先签字，可记住"以后都行" |
| 子 Agent | 派实习生去查资料，只带一页结论回来 |
| 软着陆 | 时间到交个半成品+交接说明，不摔门 |
| Provider | 插座转换头，换模型不用改电器 |

**核心思想**：模型是聪明的大脑，但它需要一套靠谱的"手脚 + 流程 + 规矩"才能真正帮你干活——这套东西就是 harness。trendpower 的每个设计，都是在让这个"大脑"既能放手做事，又不会乱来、不会卡死、不会一出错就全盘崩掉。
