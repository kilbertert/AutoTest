## 一、trendpower-tui：把核心库接到终端界面

`trendpower-tui` 用 **Textual**（Python 的终端 UI 框架，类似网页里的 React）做界面。它的职责就一句话：**界面 + 事件桥接**，真正的 agent 逻辑全在核心库里。

### 1）桥接器 AgentRunner：核心事件 ↔ 界面消息

第 2 课见过它。核心库的 `Agent.stream` 吐出来的是抽象"事件"，界面需要的是"该画什么"。`AgentRunner`（`trendpower-tui/.../tui/agent_runner.py`）就是这个翻译官，还顺手做了**性能优化**：

```python
# agent_runner.py（简化）
FLUSH_INTERVAL_SECONDS = 0.05    # 50 毫秒
async for event in self.agent.stream(user_message):
    if event["type"] == "message":
        pending.append(event["message"])
        if 距上次刷新 >= 50毫秒:
            flush()              # 批量推给界面，而不是每条都重画
    elif event["type"] == "progress":
        显示 "Thinking..." / "Running xxx..."
```

**为什么批量刷新**：模型流式输出时，一秒可能产生几十个快照。每个都重画界面会卡。**攒够 50 毫秒一批推**——肉眼觉得是实时的，机器却轻松得多。

### 2）会话持久化：`/resume` 从哪来

每完成一轮对话，transcript 存成一个 JSON 文件，下次 `/resume` 能挑出来接着聊。代码在 `trendpower-tui/.../sessions/store.py`：

```python
# sessions/store.py:save_session（简化）
payload = {"schema": 1, "id": ..., "title": 从首条用户消息提炼, "messages": messages, ...}
# 原子写：先写临时文件，再 replace —— 防止写一半崩了把存档搞坏
fd, tmp = tempfile.mkstemp(...)
写入 tmp
Path(tmp).replace(target)
```

一个**健壮性小细节**：它**先写临时文件、再原子地 `replace`**（而不是直接覆盖目标）。这样万一写到一半进程崩了，原存档还完好——不会出现"半个损坏的存档"。第 9 课的健壮性哲学，在这种边角也贯彻着。

## 二、trendpower-web：把"发给模型的内容"画到浏览器

`trendpower-web` 是个很巧的东西：它**复用整个 tui**，只额外做一件事——把**每一轮真正发给模型的请求**实时推到浏览器里可视化。这对"学习/调试上下文工程"极有价值（你能亲眼看到第 7 课讲的那些东西长啥样）。

### 它怎么做到的：继承 + 重写（provider 抽象的活教材）

还记得第 3 课说 provider 是"长得像就行"的 Protocol 吗？trendpower-web 就**继承真 provider、重写 invoke/stream**，在发请求前把请求内容广播出去：

```python
# trendpower-web/.../instrumented_providers.py（简化）
class OpenAIWithCapture(OpenAIModelProvider):      # 继承真的
    async def stream(self, params):
        request_params = self._base_params(params)  # ★这就是"发给模型的全部内容"★
        self._broadcaster.publish({"type": "llm_request", "payload": request_params})  # 广播出去
        response = await self._client.chat.completions.create(**request_params)
        async for chunk in response:
            ...
            self._broadcaster.publish({"type": "llm_response_chunk", ...})  # 回复也广播
            yield ...
```

- `_base_params(params)` 就是 provider 组装好的、**即将发给模型的那个请求字典**——也就是第 7 课说的"那桌纸"。
- `broadcaster`（`broadcaster.py`）+ aiohttp 服务器（`server.py`）把它推到浏览器前端实时显示。

**设计合理性**：因为核心库把"组装请求"封装在 provider 的 `_base_params` 里，trendpower-web 只要在 provider 那一层"插一脚"，**完全不用碰循环、工具、中间件**。这正是第 2 课"单向依赖 + 清晰边界"的红利——**想加一种全新的观测产品，只在最薄的那层切一刀就行。**

---

## 三、④ 小结

- **trendpower-tui** = 界面 + 桥接。`AgentRunner` 把核心事件翻译成界面消息，并 **50ms 批量刷新**防卡；会话**原子写**存盘支持 `/resume`；豆包等厂商是 `model_providers.py` 里的**预设**，复用 OpenAI provider。
- **trendpower-web** = 复用 tui + 在 provider 层**继承重写**，把"发给模型的请求"广播到浏览器可视化。
- 两个前端都**没碰核心逻辑**——这就是单向依赖和清晰边界换来的：核心稳定，外壳随意换、随意加。
