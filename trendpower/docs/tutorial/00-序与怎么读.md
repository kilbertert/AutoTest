# 第 0 课 · 序：这套教程怎么读

欢迎。这套教程会带你**从零读懂 trendpower 的全部设计与代码**——哪怕你现在对"AI agent"和 Python 的异步都没什么概念。

---

## 一、这套教程是写给谁的

- 你会一点 Python 的基本语法（变量、函数、类、字典），但**不熟悉异步（async）、生成器（yield）这些**——没关系，第 1 课专门补。
- 你想搞清楚的不只是"代码长什么样"，更是"**为什么这么设计**"。
- 你能接受"先慢后快"：前面打地基会细一点，地基稳了后面就快了。

---

## 二、两条贯穿全程的约定

### 约定 1：先设计，后代码

每讲一个东西，我都按同一个顺序：

> ① **要解决什么问题**（动机，常配一个生活比喻）
> ② **设计上怎么决定的**（为什么这么做，放弃了什么）
> ③ **看代码怎么实现的**（贴关键代码，逐行解释）
> ④ **一句话小结**

**为什么这样**：直接糊一脸代码，你会"每行都看得懂、合起来不知道在干嘛"。先知道"它想解决什么、打算怎么解决"，再看代码，每一行才有归属感。

### 约定 2：比喻先行

抽象概念我都先用一个生活场景打比方（贯穿全教程的两个核心比喻是：**"模型是大脑、trendpower 是手脚和规矩"** 和 **"模型每走一步都要把一整桌纸重读一遍"**），比喻立住了再上术语。

## 三、学习地图（从浅入深）

教程分两大部分，**建议按编号顺序读**，每一课用到的概念，前面一定讲过。

### Part A — Python 基础（只讲 trendpower 用得到的）

| 课 | 学完你会懂 |
|---|---|
| **01** | 异步、asyncio、生成器/yield、三种描述数据的方式、Protocol、闭包、文件与异常 —— 看 trendpower 代码不再被语法挡住 |

> Part A 不是 Python 全教程，**只讲够看懂 trendpower 的那部分**，每个概念配一个能跑的小例子，再告诉你"它在 trendpower 哪里用到"。

### Part B — trendpower 设计与代码（每课"先设计后代码"）

| 课 | 主题 | 学完你能回答 |
|---|---|---|
| **02** | 鸟瞰 + 全景路径 | 一次提问从进到出，经过哪几大站？ |
| **03** | 核心名词（foundation 层） | 系统里流动的"基本粒子"有哪几种？ |
| **04** | ReAct 循环（心脏） | agent 凭什么能自己一步步把活干完？ |
| **05** | 中间件（配件槽） | 想给 agent 加个新行为，插在哪？ |
| **06** | 工具系统 | 一次工具调用，从校验到回传走了哪几道工序？ |
| **07** | 上下文工程 ⭐ | 同一个模型，凭什么 trendpower 更便宜、更快、更稳？ |
| **08** | 能力扩展（技能 / 子 agent / MCP） | 怎么给 agent 加技能、派子任务、接外部工具？ |
| **09** | 健壮性与收尾 | 哪些地方会出错？是崩、是兜底、还是优雅收尾？ |
| **10** | 前端与周边 | 同一核心库怎么变成两种产品？ |
| **11** | 设计哲学 + 扩展指南 | 换我来加功能，从哪下手、会不会破坏分层？ |

第 **07 课（上下文工程）** 是含金量最高的一课，但它依赖前面 02–06 全部铺垫到位，所以请耐心走到那里。

```
浅  Part A ─ 01 Python 基础           ← 先把语法地基打好
   ┌─ 02 鸟瞰 + 全景                 ← 先看清骨架
```

## 六、约定的代码引用格式

文中引用代码用 **`文件路径:符号名`**，例如 `agent/agent.py:Agent.stream`。
- 路径都相对于 `trendpower-py/trendpower/`（核心库），前端会特别标注 `trendpower-tui/...`。
- 用**符号名**（函数/类名）而不是行号，因为行号会随改动变化，符号名不会。

准备好了就翻到下一课 👉 [01 · Python 基础](01-python基础.md)。
