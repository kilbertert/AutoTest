【Stage 4：每条 case - 用 claim-next 严格白名单】
循环（直到 claim-next 返回 case=null）：
  1. 抢下一条：bash: uv run python qumall-db/cli.py claim-next --db blueprints/qumall-pilot.db --worker {WORKER_ID} --lease-minutes 30
     → 返回 {"case": {...}, "remaining": N, "lease_until": "..."}
     如果 case=null（队列空）→ Stage 5
  2. chrome-devtools 驱动（白名单）：
     - list_pages / select_page
     - navigate_page / click(uid) / fill(uid,v) / press_key
     - take_snapshot ≤ 2 次
     - evaluate_script ≤ 1 次
     - take_screenshot（仅失败时）
  3. 对比 case.expected 判定 通过/失败/跳过
  4. 写回两处（必须都成功，--worker 必填避免覆盖其他 worker 的活）：
     a. bash: uv run python qumall-db/cli.py set --db blueprints/qumall-pilot.db --id <case.id> --status <S> --note "<≤200字>" --worker {WORKER_ID}
        如果返回 ok=false 且 error 包含 "claimed by worker" → 该 case 被其他 worker 抢走，本 worker 跳过这 case，**不要再写**
     b. excelio__update_cells(path=blueprints/qumall-pilot.xlsx, sheet=1, updates=[
          {row: <row>, col: 14, value: <S>},
          {row: <row>, col: 15, value: <note>},
        ])
  5. 如果本 case 执行过程崩了（浏览器错 / 超时）→ 调 release 让其他 worker 能抢：
     bash: uv run python qumall-db/cli.py release --db blueprints/qumall-pilot.db --id <case.id> --worker {WORKER_ID}
  6. UI：report_progress(done=<stats.by_status["通过"]+stats.by_status["失败"]+stats.by_status["跳过"]>, total=<stats.total>, failed=<stats.by_status["失败"]>, module=case.module)
  7. 模块最后一条结束：report_module_status(module=case.module, state="passed" or "failed")

【Stage 5：汇总】
bash: uv run python qumall-db/cli.py stats --db blueprints/qumall-pilot.db
assistant_final 报告：
- 本 worker 处理的 case 数（从 stats.by_worker 找自己）

- 然后自然结束（不要 set / claim / 等）
